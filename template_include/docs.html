03.2 템플릿 인크루드

인크루드 기능을 이용하여 템플릿를 분리하여 처리를 합니다.

03.2.1 인크루드

전처리기 일종인 인쿠르드(include)에 대해서 알아 봅니다.

php의 인크루드는 소스코드 안에서 다른 소스코드를 삽입할 수 있습니다.

예제코드: index.php

<?php

echo "안녕하세요<br>";

include "hello.php";

위와 같이 `index.php` 소스코드는 `hello.php` 소소코드를 삽입하여 결과를 출력합니다.

hello.php 코드를 한번 살펴 봅시다.

오늘은 <?= date("Y-m-d"); ?> 입니다.

일반적인 php코드와 달리 `<?php`로 시작을 하지 않고, 텍스트 일부에 `<?=` 와 `?>`로 php 코드가 삽입되어 있는 것을 볼 수 있습니다.

그럼 웹페이지를 통하여 확인을 해봅니다. 웹서버를 실행합니다.

infoh@hojin1 MINGW64 /d/32.daelim/temclude

$ php -S localhost:8000

Listening on http://localhost:8000

Document root is D:\32.daelim\temclude

Press Ctrl-C to quit.

서버를 실행한 후에 브라우저로 접속을 해봅니다.

2줄이 출력되었습니다. 첫번째 줄은 `index.php` 파일에서 출력되는 내용이고, 두번재 줄은 결합된 `hello.php` 에서 출력되는 내용 입니다.

중복실행

php의 인크루드는 2가지 종류의 include를 제공합니다.

  • include
  • include_once

인크루드는 분리되어 있는 코드조각으로 필요한 곳에 언제든지 불러와서 결합을 할 수 있습니다.

다음과 같이 `hello.php` 파일을 두번 불러와서 결합해 봅니다.

02/index.php

<?php

echo "안녕하세요<br>";

include "hello.php";

include "hello.php";

결과를 확인해 봅니다.

`hello.php` 파일을 두번 결합하여 결과가 실행되는 것을 볼 수 있습니다.

이번에는 include를 include_once로 변경을 해보도록 합니다.

03/index.php

<?php

echo "안녕하세요<br>";

include_once "hello.php";

include_once "hello.php";

변경된 코드를 다시 한번 확인을 해보도록 합니다.

소스코드에서는 `hello.php` 파일을 두번 결합을 했지만, 실제 결과는 한번만 출력 됩니다.

include_once는 두번째 결합되는 파일은 중복을 제외하기 때문에 이후 선언된 동일한 인크루드는 제외 됩니다.

분리된 코드실행

php와 같은 언어에서 include를 자주 사용하는 이유는  간단하게 코드를 분리하여 결합을 할 수 있다는 것입니다.

또한, 순수한 텍스트, html코드를 불러와 결합할 수도 있습니다.

그리고 삽입하는 코드안에 php 스크립트가 있다면 이를 해석처리를 같이 합니다.

hello.php

오늘은 <?= date("Y-m-d"); ?> 입니다.

`hello.php` 파일은 일반 택스트와 php 코드가 삽입되어 있습니다. php 코드는 서버의 날짜를 읽어서 지정한 포맷으로 출력을 합니다. 즉, php 코드가 같이 실행되는 것입니다.

변수 전달하기

인크루드로 파일을 결합할때 변수의 접근을 같이 할 수 있습니다.

`hello.php` 파일은 날짜를 출력할때 “Y-m-d” 포맷으로 출력을 합니다. 출력 포맷을 변수화 하여 가변적으로 처리를 합니다.

오늘은 <?= date($format); ?> 입니다.

`$format` 변수는 `hello.php` 파일 외에서 다음과 같이 지정을 할 수 있습니다.

<?php

echo "안녕하세요<br>";

$format="Y-m-d"; // 날짜 포맷 지정

include "hello.php";

include "hello.php";

출력 결과를 확인해 봅니다.

`index.php`에서 선언한 `format`의 변수값이 결합된 `hello.php` 파일에도 동일하게 적용이 되는 것을 확인할 수 있습니다.

`hello.php` 파일은 두번 결합되어 있습니다. 중간에 `format`의 변수값을 변경해 봅니다.

<?php

echo "안녕하세요<br>";

$format="Y-m-d"; // 날짜 포맷 지정

include "hello.php";

$format="Y-m-d H:i:s"; // 날짜 포맷 변경

include "hello.php";

결과를 확인해 봅시다.

첫번째 파일 결합은 날짜 포맷이 “Y-m-d”로 적용이 되었습니다. 두번째 파일 결합은 “Y-m-d H:i:s”로 시분초도 같이 출력되게 변경을 할 수 있습니다.

유효범위 충돌

위의 설명으로 유추할 수 있듯이 인크루드로 파일을 결합할때 변수의 범위는 글로벌 변수 접근이 됩니다.

다음과 같이 두번째 결합되는 파일 명을 변경해 봅시다.

<?php

echo "안녕하세요<br>";

$format="Y-m-d"; // 날짜 포맷 지정

include "hello.php";

$format="Y-m-d H:i:s"; // 날짜 포맷 변경

include "time.php";

include "hello.php";

변경된 `time.php` 파일은 다음과 같습니다.

현재의 날짜시각은 <?= date($format); ?> 입니다.

출력 결과를 확인해 봅시다.

세번 결합된 인크루드가 적용이 됩니다.

첫번째 결합은 Y-m-d가 적용됩니다. 두번째는 변경된 파일은 수정된 포맷 Y-m-d H:i:s 변수값을 참조 합니다. 세번째 결합은 수정되어 있는 상태 그대로Y-m-d H:i:s 변수값이 적용 됩니다.

이처럼 인크루드에서 변수명이 동일할때 서로 충돌이 발생됩니다. 충돌을 방지하기 위해서는 지역변수를 사용하는 것입니다.

지역변수

---

c언어 또는 java처럼 함수나 객체에서 시작하는 코드와 달리, 자바스크립트, python,  php와 같은 스크립트 언어에서는 소스코드의 시작이 첫줄에서 동작을 합니다.

이처럼 첫출서 부터 시작되는 언어는 기본 변수의 선언이 글로벌 변수 입니다. 또한 이때 결합된 인크루드 파일도 글로벌 변수의 접근을 사용합니다. 글로벌 변수는 다양한 충돌을 발생하는 요인이 됩니다. 이를 방지 하기 위해서 변수의 접근을 제한할 수 있는 지역변수를 사용합니다.

지역변수는 동일한 변수명이라고 해도 서로 다른 변수로 취급되기 때문에 충돌을 하지 않습니다. 새로운 메모리 영역이 추가로 생성되는 것과 같습니다.

지역변수를 선언하는 방법은 함수를 사용하는 것입니다. 다음과 같이 함수를 만들어 봅시다.

07/index.php

<?php

echo "안녕하세요<br>";

function hello($format)

{

    echo "오늘은". date($format). "입니다.";

}

$format="Y-m-d"; // 날짜 포맷 지정

hello($format);

결과를 확인해 봅시다.

소스코드에는 여러 `$format` 변수를 선언하고 사용을 하고 있습니다. 하지만, 이름만 같을뿐 서로 다른 변수 입니다.

외부에 선언된 $format은 새로운 변수를 선언하는 것입니다. 선언된 변수를 hello() 함수의 인자값으로 전달합니다. 이때 두 $format 변수는 동일한 메모리의 위치 입니다.

반면, 함수로 변수값이 전달 받을때 $format은 서로 다른 변수가 됩니다. 함수를 호출할때 지역변수가 생성되며, 지역변수로 값을 복사하게 됩니다. 즉, 복제된 서로 다른 변수가 만들어 지게 되는 것입니다.

함수 내부에서는 복제된 지역변수를 사용하게 됩니다.

함수의 변수 유효범위의 차이를 통하여 동일한 변수명을 가져도 충돌을 방지 할 수 있습니다.

예제 코드를 변경하여 실습을 해보도록 합니다.

<?php

echo "안녕하세요<br>";

function hello($format)

{

    include "hello.php";

}

$format="Y-m-d"; // 날짜 포맷 지정

hello($format);

이처럼 함수를 이용하게 되면, 글로벌 변수 `$format` 과 함수내의 `$format`은 분리가 됩니다. 서로 충돌이 발생하지 않습니다.

템흘릿으로 활용하기

인크루드 특징에 대해서 알아 보았습니다. 특징을 고려하여 인크루드를 html 템플릿으로 변경하여 사용을 해보는 코드를 작성해 봅니다.

<?php

$msg = "안녕하세요. {{hello}} 감사합니다. <br>";

function hello($format)

{

    include "hello.php";

}

$format="Y-m-d"; // 날짜 포맷 지정

$msg = str_replace("{{hello}}", hello($format) ,$msg);

echo $msg;

결과를 확인해 봅시다.

출력 내용의 순서를 확인해 봅시다.

우리의 의도는 $msg 문자열에서 {{hello}} 코드를 hello() 함수의 결과물로 변경을 하고자 하는 것입니다. 하지만, 먼저 hello() 함수가 인크루드된 파일을 먼저 출력하고, 이후에 문자열 변경을 통하여 $msg 를 출력하게 됩니다.

따라서, 우리가 원하는 순서대로 화면이 출력되지 않습니다. 인크루드 기능이 매우 유용하지만, 절차적 순서에 따라서 결과물이 출력되기 때문에, 바로 템플릿 코드로 사용하기 어렵습니다.

이를 위해서 http 출력 버퍼링을 사용합니다.

출력버퍼

php는 서버에서 브라우저로 데이터를 반환할때 중간 버퍼가 존재합니다. echo 와 print는 버퍼를 통하여 브라우저로 결과를 전송하는 것입니다.

인크루드를 템플릿으로 사용하기 위해서는 출력버퍼를 조작하여 출력 순서에 구애받지 않고 사용를 할 수 있도록 합니다.

ob_start();

ob_start()는 echo, print 출력 결과물을 바로 브라우져에 보내지 않고, 임시 버퍼로 전달을 하도록 설정을 하게 됩니다. 그리고 bo_get_clean()은 버퍼의 내용을 가지고 오고, 퍼버를 정리 합니다.

<?php

$msg = "안녕하세요. {{hello}} 감사합니다. <br>";

function hello($format)

{

    ob_start();

    include "hello.php";

    return ob_get_clean();

}

$format="Y-m-d"; // 날짜 포맷 지정

$msg = str_replace("{{hello}}", hello($format) ,$msg);

echo $msg;

즉 ob_start와 ob_get_clean을 같이 사용하면, 브라우저 출력을 하지 않고 임시 버퍼만을 이용하여 인크루드의 결과물을 반환 처리 받을 수 있습니다.

결과를 확인해 봅니다.

출력 버퍼를 조작하여 우리가 원하는 내용을 $msg 문자열 안에 넣을수 있게 되었습니다.

변수값 전달

인크루드에서 변수의 충돌을 방지하기 위해서 지역변수 변환을 하였습니다. 인크루드에서 여러개의 변수를 사용하는 경우 이를 모두 매개변수로 전달을 할 수 없습니다.

만일 매개변수를 이용한다면, 각각의 인크루드 파일마다 매개변수가 다른 함수를 선언을 해야 될 것입니다.

이를 해결하기 위해서 php의 extract 함수를 사용합니다. extract 함수는 연상배열의 키값을 변수명으로 하여 일반 변수로 물어내는 기능을 처리합니다.

다음과 같이 코드를 변경해 봅시다.

<?php

$msg = "안녕하세요. {{hello}} 감사합니다. <br>";

function hello($args=[])

{

    extract($args); // 배열을 분리

    ob_start();

    include "hello.php";

    return ob_get_clean();

}

$format="Y-m-d"; // 날짜 포맷 지정

$msg = str_replace("{{hello}}", hello(['format'=>$format]) ,$msg);

echo $msg;

hello 함수는 매개변수를 배열로 하나만 전달 받습니다. 전달되는 배열은 연상배열을 가지고 있습니다.

hello 함수는 전달받은 배열을 extract로 풀어서 인크루드 파일에서 사용을 할 수 있도록 접근을 제공합니다. 그리고 함수의 실행이 중단되면, 지역 변수들을 자동으로 삭제가 됩니다.
